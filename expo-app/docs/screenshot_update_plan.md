# Screenshot Update Plan — App Store

PropFolio now has **live in-app deal and confidence scoring**. App Store (and any marketing) screenshots can truthfully show score and confidence screens. This plan outlines what to capture and what to avoid.

## 1. Screens that can be shown

### Portfolio list

- **What to capture:** List of saved properties with visible **Deal** and **Confidence** badges on each row (e.g. "Deal 72", "High conf").
- **Include:** The one-line disclaimer below the subtitle: *"Scores are model outputs for informational use only and are not investment advice."*
- **Truthful:** Yes — scores are computed and displayed in-app.

### Property detail

- **What to capture:**
  - Hero with address, one-line deal summary, and badges (Deal X/100, confidence band).
  - Two score cards side by side (deal score + band; confidence score + band).
  - Optional: cap note (if testing with low-confidence data).
  - The one-line disclaimer under the score cards.
  - Key metrics grid (purchase price, rent, cash flow, cap rate, etc.).
  - Optional: expanded "Factor breakdown" or "About these scores" (Pro).
- **Truthful:** Yes — all from live analysis.

### Home / Get started

- **What to capture:** Title "PropFolio," subtitle "Should I buy this property?," free imports indicator, "Add property" CTA.
- **Truthful:** No score claims; safe for store.

### Paywall (optional)

- **What to capture:** Headline and benefits that reference **live** scoring (e.g. "Deal and confidence scores on every property"; "scores are for informational use only").
- **Truthful:** Benefits no longer say "coming soon."

## 2. What to avoid

- **Do not** use mockups that show scores that don’t exist in the app (e.g. different scale or labels).
- **Do not** include text that implies "coming soon" for scores or analysis.
- **Do not** imply guaranteed returns, future performance, or investment advice in screenshot text or overlays.

## 3. Recommended screenshot set (minimum)

1. **Portfolio list** — 2–3 properties with deal/confidence badges and the score disclaimer visible.
2. **Property detail** — One property with score cards, disclaimer, and key metrics visible.
3. **Home** — "Should I buy this property?" and Add property CTA.

Optional: Property detail with factor breakdown or accordion expanded (Pro); paywall with updated benefits.

## 4. Technical notes

- Use a **real** build (not a placeholder app) so scores and labels match production.
- Prefer **light and dark** variants if the app supports both, for accessibility and store variety.
- Ensure the **one-line disclaimer** is readable (not cropped) in any screenshot that shows scores.

## 5. Checklist before submission

- [ ] All screenshots show only UI that exists in the submitted build.
- [ ] No "coming soon" or placeholder score text in frame.
- [ ] Score disclaimer visible wherever scores are visible in the shot.
- [ ] No guarantee or investment-advice language in screenshot or overlay text.

PropFolio is ready for **truthful** App Store screenshots that highlight live deal and confidence scoring.
